GitHub Pages 教程（Interfit 实战版）

最后更新：2026-02-18

1. 什么是 GitHub Pages

GitHub Pages 是 GitHub 提供的静态网站托管服务，可以把仓库中的 HTML/CSS/JS 文件直接发布成公开网页。

它非常适合应用支持页、隐私政策页、FAQ 和文档网站。对需要在 App Store Connect 提供隐私政策 URL 的应用来说，这是一个低成本、上线快、可版本管理的方案。

2. 开通前准备

  1. 准备一个可用 GitHub 账号。
  2. 创建好仓库（例如 support）。
  3. 仓库中有要发布的页面文件（如 index.html、privacy-policy.html）。
  4. 改动已推送到远程分支（通常 main）。

3. 创建 support 仓库

方式 A：网页创建

  1. GitHub 右上角点击 +。
  2. 选择 New repository。
  3. 仓库名填写 support。
  4. 建议选择 Public（便于 App Store 审核访问）。

方式 B：命令行创建（gh）

git init
git add .
git commit -m "Initial support site"
gh repo create support --public --source=. --remote=origin
git branch -M main
git push -u origin main

4. 开通 GitHub Pages（网页操作）

  1. 进入仓库主页，例如 https://github.com/<username>/support。
  2. 打开 Settings。
  3. 左侧点击 Pages。
  4. Source 选择 Deploy from a branch。
  5. Branch 选择 main，Folder 选择 / (root)。
  6. 点击 Save。

保存后通常 1 到 5 分钟内可访问。

5. 开通 GitHub Pages（CLI 可选）

gh api --method POST repos/<username>/support/pages \
  -f 'source[branch]=main' \
  -f 'source[path]=/'

查看状态：

gh api repos/<username>/support/pages

返回中 status 为 built 代表构建完成。

6. Pages 地址规则

默认格式：

https://<username>.github.io/<repo-name>/

如果隐私政策文件是 privacy-policy.html，地址通常是：

https://<username>.github.io/<repo-name>/privacy-policy.html

7. 如何验证是否成功

可直接浏览器打开，或用命令行检测：

curl -I https://<username>.github.io/<repo-name>/privacy-policy.html

返回 HTTP 200 即可用于 App Store Connect。

8. 常见问题

页面 404

  • 检查 Pages 是否开启，分支和目录是否正确。
  • 确认文件路径与大小写准确。
  • 首次构建等待几分钟再访问。

推送失败（SSH 公钥）

git remote set-url origin https://github.com/<username>/support.git
git push -u origin main

文件更新后页面没刷新

  • 确认改动已提交并推送到 Pages 所用分支。
  • 强制刷新浏览器缓存。
  • 用 gh api repos/<username>/support/pages 查看状态。

9. App Store Connect 填写建议

  1. 打开 App Store Connect 的应用信息页。
  2. 找到 Privacy Policy URL。
  3. 填写可公开访问的 privacy-policy.html 链接。

建议保持链接稳定，不随版本频繁改路径。
